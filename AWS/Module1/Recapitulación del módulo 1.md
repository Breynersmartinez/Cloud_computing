# Resumen del Módulo 1

## ¿Qué es un proyecto?

Un proyecto es algo que queremos lograr o completar.

Ejemplos:
- Construcción de una autopista  
- Lanzamiento de un satélite  
- Desarrollo de software  

Todo proyecto requiere:

### Recursos
- Presupuesto  
- Materiales  
- Personas  

### Requisitos
- Definen qué se espera lograr  
- Permiten saber cuándo el proyecto está terminado  

Un proyecto finaliza cuando cumple con sus requisitos.

---

## Aplicación al software

Un proyecto de software también necesita:

- Recursos  
- Requisitos  

Para gestionarlo correctamente se utiliza el **SDLC (Software Development Life Cycle)**.

---

## Etapas principales del SDLC

### 1. Análisis
- Definir el problema que se quiere resolver.
- Evaluar qué ocurre si no se realiza el proyecto.
- Justificar su necesidad.

### 2. Diseño
- Definir detalles técnicos.
- Planear cómo se construirá la solución.

### 3. Implementación
- Desarrollo y construcción del software.
- Evitar comenzar aquí sin análisis y diseño previos.

### 4. Pruebas
- Validar que la aplicación funciona como se esperaba.
- Confirmar que es estable para producción.
- Las pruebas pueden superponerse con otras fases.
